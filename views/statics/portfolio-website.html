Portfolio Website

Background

I came to the idea of making my own website when I was looking for designer and developer job. Most of the time in Japan, you have to write job application for every single each company you apply for which I a pretty tiring process. I think it would be better that you write whatever you want you potential employers to know about you on your website. Moreover, most designers have their own websites I actually already had my first version website, which I think suck. That’s why I decided to make a new one.

Problem with the first portfolio

The problem with the first portfolio is that

  1. There are no stories behind them, I just put all of the designs in the website, few texts, no stories.
  2. It doesn’t look so professional, due to the theme the layout and most importantly the url, I used heroku at that time becasue it's free
  3. It is mostly static endnote responsive. I would like it to be more interactive. Some animations, for example.

Of course, there is a good point I like about this version too which is the fact that it is a single page application and the navigation is really intuitive.

Objective

In addition to the above problem of my first version website, the reasons I am redesigning my new website is

  • Firstly to showcase my design and developing skills.
  • Doing passive job searching.
  • Get hands-on experience for building proper website
  • Commit myself to finish a project thoroughly

Result

The result is obviously this website !

Basically everything. Design including visual design, user experience and coding.

Reflection

Good points

  • This website includes case studies so employers know how I think
  • It looks kinda professional in my opinion

Bad points

  • I took a lot of time, like few weeks even though I planned to finish it in few days. I did this during my trips to Thailand, Myanmar, and Kansai area in japan
  • This website does look professional, however, it does really tell about my personality. I am a more fun person. I think a portfolio website should tell about that person personality too.
  • I did everything mixed. Like one day I would do the backend, I would change to front end even thoughI didn’t finish the backend yet. That is why I would like to learn more about the development workflow in a more systematic way
